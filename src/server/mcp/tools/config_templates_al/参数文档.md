# `config_templates_al` 参数文档

## 文档范围

本文档说明 `examples/framework_demo/config_templates_al` 目录下 7 类任务参考配置的参数意义、单位、边界条件和推荐用法，供同事交接时直接使用。

对应数据库：

- `database/Al-Si-Mg-Fe-Mn_by_wf.TDB`

当前目录包含的 7 类任务：

1. `point_calculation`
2. `binary_equilibrium`
3. `line_calculation`
4. `ternary_calculation`
5. `thermodynamic_properties`
6. `scheil_solidification`
7. `boiling_point`

当前这 7 个模板均已实际回归运行通过。

## 一、数据库特点与模板选型

### 1. 数据库组元

该数据库的主体元素为：

- `AL`
- `SI`
- `MG`
- `FE`
- `MN`

代码内部会自动补入：

- `VA`

因此：

- 二元任务选用 `AL-SI`，用于稳定生成典型铝硅相图。
- 三元任务选用 `AL-MG-SI`，用于覆盖铝合金中最常见的析出/共晶子系统。
- 其余任务使用 5 元 Al-rich 组成，以显式保留 Fe/Mn 相关金属间化合物。

### 2. 当前默认 5 元示例成分

默认示例成分采用摩尔分数：

- `AL = 0.93`
- `SI = 0.04`
- `MG = 0.01`
- `FE = 0.015`
- `MN = 0.005`

约束：

1. 各组元分数必须在 `[0, 1]` 内。
2. 所有显式组元之和应为 `1.0`。
3. 在内部求解时，通常会把一个组元当作依赖变量，因此日志里不一定显示所有元素的 `X_*` 变量，这是正常现象。

### 3. 当前推荐激活相集合

当前多元任务中使用的推荐相集合为：

- 基体/液相：`LIQUID`, `FCC_A1`, `DIAMOND_A4`
- Mg-Si/Mg-Al 相关：`HCP_A3`, `BETA_ALMG`, `EPSILON_ALMG`, `GAMMA_ALMG`, `MG2SI`
- Fe/Mn/Si 相关：`BCC_A2`, `CBCC_A12`, `AL5FE2`, `AL13FE4`, `ALPHA_ALFESI`, `BETA_ALFESI`, `ALPHA_ALFEMNSI`, `AL4_FEMN`

建议：

- 不要一开始把数据库中 50 多个相全部激活。
- 先用当前这套相列表做基础分析，再根据具体问题添加相。

## 二、通用字段说明

### 1. 顶层字段

所有任务通常都包含以下顶层字段：

| 字段 | 含义 | 说明 |
| --- | --- | --- |
| `task_type` | 任务类型 | 必填，决定 `Framework` 调用哪类算法 |
| `tdb_file` | 数据库路径 | 必填，当前统一为 `database/Al-Si-Mg-Fe-Mn_by_wf.TDB` |
| `task_name` | 任务名 | 建议唯一，便于结果目录辨识 |
| `task_path` | 结果输出目录 | 建议放在 `examples/framework_demo/result/...` |
| `condition` | 任务条件 | 必填，真正的计算参数都在里面 |

### 2. `condition.components`

含义：

- 参与计算的元素列表，不写 `VA`，代码会自动补。

边界：

- 只能写数据库中存在的元素。
- 当前数据库安全使用范围是 `AL / SI / MG / FE / MN`。

建议：

- 二元任务只放 2 个元素。
- 三元任务只放 3 个元素。
- 多元点计算、Scheil、线扫描、性质计算再放 5 个元素。

### 3. `condition.activated_phases`

含义：

- 明确允许参与平衡竞争的相。

边界：

- 相名必须存在于数据库中。
- 写得越多，计算越慢，且更容易出现不必要的相竞争。

建议：

- 优先保留与问题直接相关的相。
- 对 Fe/Mn 富化的再生铝问题，建议保留 `AL13FE4`、`AL5FE2`、`ALPHA_ALFESI`、`BETA_ALFESI`、`ALPHA_ALFEMNSI`、`AL4_FEMN`。

## 三、7 类任务逐项说明

## 1. `point_calculation`

### 作用

在固定温度、固定组成下做一次平衡计算，用于查看稳定相、相分数和单点状态。

### 当前模板参数

| 参数 | 当前值 | 单位 | 含义 |
| --- | --- | --- | --- |
| `components` | `AL, SI, MG, FE, MN` | - | 参与平衡的元素 |
| `activated_phases` | 16 个相 | - | 允许竞争的相集合 |
| `temperature` | `850` | K | 计算温度 |
| `compositions` | `0.93/0.04/0.01/0.015/0.005` | 摩尔分数 | 合金组成 |

### 边界与建议

- `temperature` 必须大于 `0`，工程上建议在 `500-1100 K` 内先测试。
- `compositions` 总和必须为 `1.0`。
- 如果目标是看铸态终凝组织，`800-900 K` 是较好的起点。

### 输出

- 单点平衡结果目录
- 常见会包含表格型结果文件

## 2. `binary_equilibrium`

### 作用

计算二元相图。

### 当前模板参数

| 参数 | 当前值 | 单位 | 含义 |
| --- | --- | --- | --- |
| `activated_elements` | `AL, SI` | - | 二元相图元素 |
| `components` | `AL, SI` | - | 参与计算的元素 |
| `activated_phases` | `LIQUID, FCC_A1, DIAMOND_A4` | - | 二元相图中的主要相 |
| `compositions_start` | `AL=1.0, SI=0.0` | 摩尔分数 | 起始端点 |
| `compositions_end` | `AL=0.7, SI=0.3` | 摩尔分数 | 终止端点 |
| `temperature_start` | `500` | K | 起始温度 |
| `temperature_end` | `1200` | K | 终止温度 |

### 边界与建议

- 二元相图中两元素分数分别处于 `[0, 1]`，并满足总和为 `1.0`。
- 当前模板只扫描 `0-30 mol% Si`，偏向 Al-rich 区域。
- 若想看接近共晶附近，可继续加密 `SI` 在 `0.08-0.18` 附近的范围。

### 实现细节

- 当前运行日志显示内部会自动转成固定步长扫描。
- 当前这组模板实际生成了 3 个相的边界，适合作为基础示例。

### 输出

- `binary_equilibrium.json`
- 相图相关图像或数据文件

## 3. `line_calculation`

### 作用

在固定组成下沿温度方向扫描，输出相状态与化学势等随温度变化的信息。

### 当前模板参数

| 参数 | 当前值 | 单位 | 含义 |
| --- | --- | --- | --- |
| `components` | `AL, SI, MG, FE, MN` | - | 元素集合 |
| `compositions_start` | 5 元固定成分 | 摩尔分数 | 扫描使用的组成 |
| `compositions_end` | 与 `start` 相同 | 摩尔分数 | 当前实现中基本不作为独立路径终点使用 |
| `temperature_start` | `500` | K | 扫描起点 |
| `temperature_end` | `950` | K | 扫描终点 |
| `increments` | `25` | K | 温度步长 |
| `activated_phases` | 16 个相 | - | 参与竞争的相 |

### 边界与建议

- 当前任务本质上是“定成分温度扫描”，不是任意成分路径扫描。
- `increments` 建议取 `5-25 K`。
- 若关注析出起始温度，建议将步长缩小到 `5-10 K`。

### 实现细节

- 当前回归结果成功输出了 `MU(AL)`, `MU(SI)`, `MU(MG)`, `MU(FE)`, `MU(MN)`。
- 从实际输出看，上端温度点可能不是严格包含 `temperature_end`，因此不要把 `end` 机械理解为一定落点。

### 输出

- 线扫描结果表
- 常见包括化学势、相分数等信息

## 4. `ternary_calculation`

### 作用

计算三元等温截面。

### 当前模板参数

| 参数 | 当前值 | 单位 | 含义 |
| --- | --- | --- | --- |
| `components` | `AL, MG, SI` | - | 三元系统 |
| `activated_phases` | `LIQUID, FCC_A1, DIAMOND_A4, HCP_A3, BETA_ALMG, EPSILON_ALMG, GAMMA_ALMG, MG2SI` | - | 三元等温截面的主要相 |
| `temperature` | `773` | K | 等温截面温度 |
| `compositions_y` | `AL=1, MG=0, SI=0` | 摩尔分数 | 三角形顶点 Y |
| `compositions_x` | `AL=0, MG=1, SI=0` | 摩尔分数 | 三角形顶点 X |
| `compositions_o` | `AL=0, MG=0, SI=1` | 摩尔分数 | 三角形顶点 O |

### 边界与建议

- 三个顶点必须分别是合法组成，且每个顶点分数和为 `1.0`。
- `773 K` 适合作为时效相关或中温截面起点。
- 如果想更贴近铸态凝固后组织，可考虑 `800-900 K`。

### 实现细节

- 当前回归运行得到 5 个相、2161 个点、218 条 tie-lines、575 个 tie-triangles。
- 这是当前数据库下比较稳定的一组三元示例。

### 输出

- `ternary_plotly.json`
- `ternary_equilibrium.png`

## 5. `thermodynamic_properties`

### 作用

在给定成分和温压范围内计算热力学性质，例如 Gibbs 能、焓、熵和定压热容。

### 当前模板参数

| 参数 | 当前值 | 单位 | 含义 |
| --- | --- | --- | --- |
| `components` | `AL, SI, MG, FE, MN` | - | 元素集合 |
| `activated_phases` | 16 个相 | - | 参与竞争的相 |
| `compositions_start` | 5 元固定成分 | 摩尔分数 | 起始组成 |
| `compositions_end` | 与 `start` 相同 | 摩尔分数 | 结束组成；当前模板做定成分性质扫描 |
| `temperature_start` | `500` | K | 起始温度 |
| `temperature_end` | `950` | K | 结束温度 |
| `increments` | `25` | K | 温度步长 |
| `pressure_start` | `5` | `log10(P/Pa)` | 起始压力 |
| `pressure_end` | `5` | `log10(P/Pa)` | 结束压力 |
| `pressure_increments` | `2` | 对数压力步长 | 压力扫描步长 |
| `properties` | `GM, HM, SM, CPM` | - | 输出性质名 |

### 性质说明

| 性质 | 含义 | 常用单位 |
| --- | --- | --- |
| `GM` | 摩尔 Gibbs 自由能 | J/mol |
| `HM` | 摩尔焓 | J/mol |
| `SM` | 摩尔熵 | J/mol/K |
| `CPM` | 摩尔定压热容 | J/mol/K |

### 边界与建议

- 当前实现中压力不是直接填 Pa，而是 `log10(P/Pa)`。
- `pressure_start = 5` 对应 `10^5 Pa = 100000 Pa`，接近常压。
- 如果只是常压性质曲线，保持 `pressure_start = pressure_end = 5` 即可。

### 实现细节

- 当前回归成功输出了 `18` 个温度点、`16` 个相、`4` 类性质。
- 运行日志显示内部实际压力为 `100000 Pa`，验证了对数压力解释。

### 输出

- `thermodynamic_properties.json`
- 表格型性质结果

## 6. `scheil_solidification`

### 作用

执行 Scheil-Gulliver 非平衡凝固模拟。

### 当前模板参数

| 参数 | 当前值 | 单位 | 含义 |
| --- | --- | --- | --- |
| `components` | `AL, SI, MG, FE, MN` | - | 合金元素 |
| `compositions` | 5 元固定成分 | 摩尔分数 | 初始合金组成 |
| `start_temperature` | `1100` | K | 起始温度，应高于液相线 |
| `temperature_step` | `5` | K | 降温步长 |
| `activated_phases` | 16 个相 | - | 凝固过程中允许生成的相 |
| `inhibit_phases` | `[]` | - | 需要强制屏蔽的相 |

### 边界与建议

- `start_temperature` 必须高于液相线，否则会从部分凝固状态开始。
- `temperature_step` 建议 `1-5 K`。
- 如果只关心主凝固路径，`5 K` 足够快；如果关心析出先后次序，建议用 `1-2 K`。

### 当前回归现象

当前示例在实际回归中观察到：

- `AL13FE4` 约在 `965 K` 出现
- `ALPHA_ALFEMNSI` 约在 `950 K` 出现
- `FCC_A1` 约在 `900 K` 出现
- `ALPHA_ALFESI` 约在 `895 K` 出现
- `BETA_ALFESI` 约在 `875 K` 出现
- `DIAMOND_A4` 约在 `835 K` 出现
- `MG2SI` 约在 `832 K` 出现

这说明当前模板已经能反映出 Fe/Mn 富化铝合金中典型的复杂凝固相竞争。

### 输出

- `scheil_solidification.json`
- `scheil_solidification.csv`
- 相关图像文件

## 7. `boiling_point`

### 作用

搜索给定成分下的固相线、液相线、bubble point 和 dew point。

### 当前模板参数

| 参数 | 当前值 | 单位 | 含义 |
| --- | --- | --- | --- |
| `components` | `AL` | - | 当前模板使用单元素 Al |
| `pressure` | `101325` | Pa | 外压 |
| `compositions` | `AL = 1.0` | 摩尔分数 | 组成 |
| `temperature_range` | `[800, 4000]` | K | 名义搜索区间 |

### 边界与建议

- 这一类任务更适合作为纯元素或非常简单组分的演示任务。
- 对多元 Al 合金而言，bubble/dew point 的工程意义通常不如 solidus/liquidus 直接。

### 代码层面注意

- 当前实际回归中，虽然输入了 `temperature_range = [800, 4000]`，但输出中仍出现了接近 `6000 K` 的 bubble/dew point。
- 这说明当前实现里，`temperature_range` 对搜索上界的约束并不完全可靠。

### 输出

- `boiling_melting_point.csv`

## 四、交接时建议告诉同事的几个重点

## 1. 先从低维任务入手

建议顺序：

1. `point_calculation`
2. `binary_equilibrium`
3. `ternary_calculation`
4. `scheil_solidification`
5. `line_calculation` / `thermodynamic_properties`

这样更容易先建立对数据库相稳定区间的直觉。

## 2. 相列表不要一次性开满

当前数据库有 56 个相，但模板只激活与 Al-Si-Mg-Fe-Mn 问题强相关的一小部分相。

原因：

- 相开得越多，收敛和判相越复杂
- 很多冷门相会干扰结果解释

## 3. 5 元模板更适合再生铝/复杂杂质场景

如果问题偏向：

- Fe 富集杂质
- Mn 补偿
- 再生铝凝固组织

就优先使用当前 5 元模板。

如果问题偏向：

- Al-Mg-Si 析出体系
- 6xxx 近似子系统

可以先用三元模板，再逐步引入 `FE` 和 `MN`。

## 4. 当前代码实现下的两个重要细节

1. `thermodynamic_properties` 的压力字段是 `log10(P/Pa)`，不是直接 Pa。
2. `boiling_point` 的 `temperature_range` 当前不能完全视作严格搜索边界。

## 五、建议维护方式

建议后续维护时至少保留这 3 套基准：

1. `AL-SI` 二元相图模板
2. `AL-MG-SI` 三元等温截面模板
3. `AL-SI-MG-FE-MN` 五元 Al-rich 模板

这样后续无论是做基础相图、析出体系，还是做含 Fe/Mn 杂质的凝固模拟，都有稳定起点。
